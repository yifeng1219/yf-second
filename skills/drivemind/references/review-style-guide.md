# DriveMind Review Style Guide

Use this guide when writing reviews so they stay structured without sounding stiff.

## Core rule
Keep the six-part review structure, but adapt the language to the actual task.

## Good review qualities
- specific, not generic
- calm, not dramatic
- honest about limits
- focused on what changed the result
- useful for the next run

## Avoid
- empty ceremony
- repeating the same heading content with no substance
- turning every review into a dry checklist when a short narrative would be clearer
- dropping key sections entirely

## Preferred pattern
A good DriveMind review should feel like:
- a real retrospective,
- with stable structure underneath,
- and enough natural language that a human would actually want to read it.

## Minimum required structure
1. Outcome
2. What happened
3. What changed the result
4. Escalation boundary
5. Reusable lesson
6. Next-time rule
