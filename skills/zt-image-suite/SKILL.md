# zt-image-suite

这是总作图 skill。目标是：拿到需求后，尽快产出可直接发出的图片成品。

## 覆盖任务

- 参考图重做
- 海报设计
- 小红书/社媒封面图
- 宣传图 / 广告图
- 单图信息排版
- 同内容换风格
- 图片二改、三改
- 生成后正确发送图片附件

## 先做任务分类

### A. 参考图重做
适用于：
- 内容一样但排版不一样
- 参考这个图做新版
- 保留主题，换一套视觉

执行时直接按 `zt-poster-redesign` 的思路做：
- 自己理解图片内容
- 重做层级与版式
- 重做核心视觉元素
- 不照搬原图样式

### B. 从零生成宣传图
适用于：
- 用户只有主题，没有参考图
- 用户要你直接设计海报/封面/广告图

这时先补齐最少信息：
- 主题
- 风格
- 比例
- 目标平台
- 是否允许你优化文案

若用户没说全，默认按最合理方案补齐，不要来回盘问。

### C. 二改/精修
适用于：
- 再改一版
- 更高级一点
- 更像爆款封面
- 门店广告风 / 商务风 / 科技风

这时优先只改一个维度：
- 版式
- 配色
- 标题层级
- 核心视觉
- 营销感强弱

不要每次全推翻，避免返工过大。

## 标准作图流程

1. 确认最少必要信息
   - 主题
   - 风格
   - 比例
   - 是否可优化文案层级

2. 自主理解输入内容
   - 有参考图：自己提取主标题、卖点、结构、按钮区、说明区
   - 有文案：自己判断主副标题层级
   - 不要轻易把整理文案的活甩回给用户

3. 先想清楚画面骨架
   - 主标题放哪
   - 视觉主体是什么
   - 卖点怎么分区
   - CTA/按钮区怎么收口

4. 生成图片 Prompt
   Prompt 里必须交代：
   - 中文商业海报/封面图
   - 不照搬参考图版式
   - 信息层级清晰
   - 适合社交平台
   - 指定比例
   - 指定不要 mockup / 手机框 / 无关人物写真 / 杂乱背景

5. 出图后先检查结果
   - 主题是否对
   - 画风是否偏题
   - 文字区域是否足够
   - 是否像可交付成品

6. 发送前校验真实文件路径
   这是硬规则：
   - 不要默认 `MEDIA:./文件名` 就能发
   - 先检查该文件是否真实存在
   - 若不存在，去系统媒体目录找实际生成文件
   - 常见目录：
     - `C:\Users\yifeng\.openclaw\media\tool-image-generation\`
     - `C:\Users\yifeng\.openclaw\media\inbound\`
   - 找到真实文件后再发送

7. 默认直接交付成图
   - 用户说“做好了发我”，优先直接发图
   - 除非用户要解释，否则别写长篇说明

## 常用风格映射

- 海报风：强标题、强卖点、广告感明显
- 科技感：蓝白/深色、发光、几何界面、数据感
- 商务风：克制、清爽、规整、可信赖
- 小红书风：醒目封面、大字、强对比、轻营销
- 高级感：留白、简洁、少元素、色彩克制
- 门店宣传风：直给卖点、服务内容醒目、转化导向强

## Prompt 模板

### 模板 1：参考图重做

```text
设计一张中文{比例}宣传图，主题围绕“{主题}”，整体为{风格}，参考原图内容方向，但不要照搬原图版式。要求：
1）主标题醒目；
2）围绕原图业务内容重做信息层级；
3）重新设计新的核心视觉元素，不使用原图造型；
4）版式更成熟，像可直接投放/发布的商业成品；
5）包含行动引导区；
6）适合社交平台；
7）不要 mockup，不要手机外框，不要人物写真；
8）输出高质量成品。
```

### 模板 2：从零出图

```text
设计一张中文{比例}{风格}宣传图，主题是“{主题}”。要求：
1）标题层级强；
2）卖点清晰；
3）有视觉主体；
4）配色统一；
5）适合{平台}发布；
6）包含引导动作区；
7）输出成熟广告成品图。
```

## 发送规则

- 用户要图：优先直接发图片附件
- 如果第一次发送异常：立刻排查真实文件路径，再重发
- 不要连续两次发送同一个错误路径

## 输出要求

默认输出：
- 直接成图
- 一句简短说明
- 如需继续迭代，只提供少量明确选项

## 示例调用语

- 用 zt-image-suite，帮我做一张宣传海报
- 调用 zt-image-suite，参考这张图重做一版正方形封面
- 用 zt-image-suite，把这张图改得更高级一点
- 用 zt-image-suite，做一张电脑维修服务广告图
- 用 zt-image-suite，图片做好直接发我，不要只讲思路

## 禁忌

- 不要把图片理解工作先丢给用户
- 不要只给思路，不出图
- 不要照搬参考图版式
- 不要在未确认文件存在时发送 `MEDIA:` 占位路径
- 不要为了“改很多”而无意义重做，优先做有效变化
