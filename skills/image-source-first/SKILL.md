# image-source-first

收到图片需求时，默认先走“先搜索，后生成”的流程。

## 默认流程

1. 先判断是否属于“可搜索现成图”场景
   - logo / 图标 / 品牌素材
   - 官方公开 PNG / SVG
   - 透明底素材图
   - 参考图里可拆分的单个现成元素
   - 常见配图、通用素材图

2. 能搜索就优先搜索，不先生成
   - 优先找官方来源、品牌资源页、Wikimedia Commons、可信公开素材站
   - 目标是拿到“可直接发”的真实文件，不是只给链接

3. 找到候选图后，先做发送前校验
   - 检查真实文件是否存在
   - 检查文件大小是否正常，不发空文件/坏文件
   - 尽量检查图片可读取、尺寸正常
   - 确认没问题再发送

4. 搜索失败或候选图不合格时，再走重做/生成
   - 搜不到
   - 清晰度不够
   - 背景不对
   - 尺寸不符
   - 风格不符
   - 用户明确说“重做一张”

5. 生成或重做后，仍然要先校验再发送
   - 不允许把未检查的半成品直接发给用户

## 来源优先级

按这个顺序找：

1. 官方品牌资源 / 官方站点
2. Wikimedia Commons
3. 可信公开素材站
4. 自己重做 / 生成

原则：能直接拿现成可用图，就不要先自己画。

## 发送前质检清单

发送任何图片前，至少过一遍下面清单：

- [ ] 文件真实存在
- [ ] 文件大小大于 0，且不是明显异常的小空壳文件
- [ ] 文件扩展名与用户需求一致，例如 PNG / JPG / WEBP
- [ ] 图片可正常读取，不是损坏文件
- [ ] 图片尺寸正常，宽高不是 0
- [ ] 背景、透明度、比例、清晰度符合用户要求
- [ ] 内容没有发错：图标没发成截图，单图没发成合成图，透明底没发成带底图
- [ ] 路径是真实落点，不是想当然的相对路径或占位路径
- [ ] 若同一任务发多张图，确认每一张都单独检查过

只要有一项不通过：
- 立刻停止发送
- 直接打回重做或改走别的来源
- 不把半成品、坏文件、错文件发给用户

## 发送规则

- 用户要图片时，优先直接发送图片附件
- 不要只回链接，除非用户明确只要链接
- 不要在未确认文件存在时发送 `MEDIA:` 路径
- 默认先做质检，再发送
- 如果任一检查不通过，直接打回重做，不把问题文件发给用户

## 输出风格

默认简短：
- 已找到，直接发图
- 没找到，已重做后发图
- 如有必要，再补一句差异说明

## 失败兜底

如果搜索渠道拿不到合格图片：
- 直接切换到生成 / 重做
- 优先交付用户可直接使用的成品
- 不要卡在“搜不到”这一步反复空转

## 适用提醒

这个 skill 特别适合：
- “给我一个 XX logo 的 png”
- “把这个图标单独发我”
- “给我官方图标”
- “找一张能直接用的图片发给我”
- “先别做，先搜现成的”

## 禁忌

- 不要一上来就生成，忽略搜索渠道
- 不要只搜到页面，不落地文件
- 不要不检查就发送
- 不要把损坏图、空图、错误图发给用户
