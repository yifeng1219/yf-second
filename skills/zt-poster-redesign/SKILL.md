# zt-poster-redesign

按下面流程执行，默认直接交付成品，不把整理文案的工作甩回给用户。

## 核心目标

把用户给的参考图，做成一张**内容同主题 / 表达一致 / 视觉重做 / 排版不同**的新图。

默认追求：
- 中文信息层级清楚
- 视觉元素重画，不照搬原图造型
- 更像可直接发出的成品图，而不是草稿
- 尽量一次出图就可用

## 标准流程

1. 先确认最少必要信息
   - 风格：如海报风、科技感、极简、商务、小红书风
   - 比例：正方形、竖版、横版
   - 是否允许优化文案层级

2. 拿到参考图后，默认自己完成理解
   - 不要第一反应让用户手抄文案
   - 先根据图片内容自行提取主标题、卖点、按钮区、说明区、视觉主体
   - 如果个别小字无法完全识别，也要先给出可用版本，而不是把工作推回去

3. 组织重设计思路
   - 保留核心主题与业务方向
   - 重做标题层级
   - 重做视觉主体，例如磁盘/C盘/图标/卡片/按钮样式
   - 重组信息分区，让画面更像广告海报

4. 生成图片
   - Prompt 必须明确写：不要照搬原图版式
   - 指定中文商业海报、信息层级清晰、适合社交平台
   - 默认整张图不要出现英文；除非产品名、品牌名或用户明确指定内容本身就带英文，才允许保留必要英文
   - 指定 1:1 或用户要求的比例
   - 指定不要手机壳、人物写真、样机、无关背景

5. 发送前必须校验真实文件路径
   - 不要直接猜 `MEDIA:./xxx.png` 是否存在
   - 先确认生成文件的真实落点
   - 如果工具结果没有直接暴露稳定相对路径，就去系统媒体目录里找刚生成的文件
   - 只有确认文件真实存在后，才能用 `MEDIA:` 发送

6. 发图后再补一句简短说明
   - 已发成品
   - 如需要，再继续二改：更像门店广告 / 更高级 / 更像爆款封面

## 图片发送硬规则

这条是为了避免再次出现“图生成了但没发到用户那里”的问题。

- 如果先前引用的是 `MEDIA:./文件名`，发送前必须确认该文件真的在工作区存在。
- 若工作区不存在，但系统媒体目录存在生成文件，必须改发真实绝对路径。
- 常见真实目录示例：
  - `C:\Users\yifeng\.openclaw\media\tool-image-generation\`
  - `C:\Users\yifeng\.openclaw\media\inbound\`
- 用户说“图片做好了发我”时，优先直接只发图片；若发送异常，再立即排查文件路径。

## Prompt 模板

可按需替换主题词：

```text
设计一张中文{比例}宣传海报，主题是“{主题}”，整体为{风格}，不要照搬参考图版式。要求：
1）主标题醒目；
2）围绕原图业务内容重做信息层级；
3）重新设计一套新的核心视觉元素，不使用原图造型；
4）排版像成熟商业广告图，中文可读性强；
5）包含按钮式引导区；
6）适合社交平台发图；
7）不要 mockup，不要手机外框，不要人物写真；
8）输出高质量成品。
```

## 输出要求

默认给用户：
- 直接成品图
- 简短一句说明
- 如果要继续迭代，只给 2~3 个明确方向供选，不要长篇解释

## 示例调用语

- 用 zt-poster-redesign，参考这张图给我重做一版海报
- 调用 zt-poster-redesign，内容一样，但排版和 C 盘样式都换掉
- 用 zt-poster-redesign，把这张宣传图做成正方形海报风
- 用 zt-poster-redesign，保留原图主题，重做一个更高级的广告版式
- 用 zt-poster-redesign，图片做好直接发我

## 禁忌

- 不要让用户自己去抄整张图的文字，除非原图严重缺失且确实无法识别
- 不要照搬原图版式
- 不要只给思路不给图
- 不要在未确认文件存在的情况下发送占位 `MEDIA` 路径
